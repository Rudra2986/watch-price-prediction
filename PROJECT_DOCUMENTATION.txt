1. PROJECT NAME
------------------------------------------------------------
Watch Price Prediction (WatchVine Dataset)


2. PROJECT OBJECTIVE
------------------------------------------------------------
Build a machine learning regression model to predict the
price of watches (in INR) based on product attributes
scraped from the WatchVine Indian e-commerce platform.
The final deliverable is a Streamlit web application where
users can input watch attributes and receive an instant
price prediction.


3. COMPLETE PROJECT OVERVIEW
------------------------------------------------------------
This project follows a structured 7-step ML pipeline:

  Step 1 — Data Cleaning
    • Raw scraped data (3000+ columns) cleaned to 18 columns
    • Price fixed from string to float, leakage column dropped
    • Missing categoricals filled with 'unknown', text normalized

  Step 2 — Exploratory Data Analysis (EDA)
    • 25 plots generated analyzing price distribution, brands,
      categorical features, correlations, and feature quality
    • Key finding: 87.5% of watches have unknown brand
    • Key finding: Price is right-skewed, log transform needed
    • Key finding: is_automatic == ai_is_automatic (100% identical)

  Step 3 — Feature Engineering
    • Dropped 3 redundant columns (ai_is_automatic, ai_watch_type, name)
    • Created 3 binary flag columns (has_brand, has_details, has_watch_type)
    • Encoded is_automatic: yes=1, no=0, unknown=-1
    • One-Hot Encoded 10 low/medium cardinality columns
    • Target Encoded 3 high-cardinality columns (color, dial_color, strap_color)
    • Train/test split (80/20) done BEFORE encoding (leakage-free)
    • Log-transformed target: log_price = np.log1p(price)
    • Saved encoder artifacts for deployment (target_encoders.pkl, train_columns.pkl)

  Step 4 — Model Building
    • Trained 6 regression models with 5-fold cross-validation
    • Models: Linear Regression, Ridge, Random Forest, XGBoost, LightGBM, CatBoost
    • Linear/Ridge models use StandardScaler via Pipeline
    • All models saved as .pkl files

  Step 5 — Model Evaluation
    • Evaluated all 6 models on held-out test set (298 rows)
    • Metrics: R2, RMSE (log), MAE (log), RMSE (INR), MAE (INR)
    • Top 2 models identified: CatBoost (R2=0.4544) and Random Forest (R2=0.4273)
    • 5 evaluation plots generated

  Step 6 — Hyperparameter Tuning
    • Used Optuna (Bayesian optimization) with 100 trials for CatBoost
    • Best params: iterations=540, depth=5, learning_rate=0.0465
    • Final model: CatBoost with R2=0.4503, RMSE=INR 658 on test set
    • SHAP feature importance analysis performed
    • final_model.pkl and model_metadata.pkl saved

  Step 7 — Streamlit Deployment (v3 — Chronos AI Premium)
    • Futuristic AI dashboard with cyberpunk-inspired dark theme
    • Brand identity: "Chronos AI — Watch Price Intelligence"
    • Animated floating orb background (CSS-only, no dependencies)
    • Shimmer gradient hero title with Space Grotesk typography
    • KPI strip with model accuracy, watches trained, features, precision
    • 3-tab navigation: Predict | Analytics | About
    • Glassmorphism cards with hover glow effects
    • Animated price reveal with blur-to-sharp transition
    • Plotly gauge chart for prediction confidence
    • Interactive SHAP bar chart (Plotly) with purple/red color coding
    • AI Insights cards (contextual recommendations)
    • Analytics dashboard: model comparison chart, pipeline overview
    • Simplified input: 10 fields (4 redundant auto-derived)
    • Premium micro-interactions and smooth animations throughout
    • Mobile-responsive design

11. IMPORTANT DECISIONS MADE
------------------------------------------------------------
1. 'unknown' kept as valid category (NOT imputed)
   Reason: 57-89% of data is unknown. Imputing would inject fake data
   into the majority of rows.

2. Log transform on price (target variable)
   Reason: Price is right-skewed. log1p(price) makes it more normal,
   improving model performance and reducing outlier influence.

3. Train/test split BEFORE encoding
   Reason: Target encoding uses price statistics. Encoding before
   splitting would leak test information into training.

4. is_automatic encoded as 1/0/-1 (not 1/0)
   Reason: -1 keeps "unknown" distinct from "definitely not automatic".
   Tree-based models handle -1 naturally.

5. CatBoost selected as final model
   Reason: Best R2 on both cross-validation (0.3768) and test set
   (0.4503 after tuning). Handles categorical features well.

6. StandardScaler applied only to Linear/Ridge models
   Reason: Tree-based models (XGBoost, LightGBM, CatBoost, RF) are
   scale-invariant by design and don't need scaling.

7. random_state=42 used everywhere
   Reason: Reproducibility across all steps and models.

8. Streamlit inputs reduced from 14 to 10
   Reason: Data analysis showed 4 fields are redundant:
   - gender: derivable from category (men's watch = men)
   - watch_style_category: 100% identical to watch_type
   - color: 95.6% identical to dial_color
   - material: 97.5% identical to strap_material
   Auto-deriving these reduces user effort without losing accuracy.

9. Brand name and movement status extracted from watch titles
   Reason: 87.5% of brands and 89.1% of movements were unknown in the cleaned data,
   causing a performance bottleneck. Extracting these from watch names reduced
   brand sparsity to 0.07% and recovered 300+ automatic movement watches, raising R2 to 71.5%.

10. Dynamic selectbox options in Streamlit App
    Reason: To support 62 unique brand names and other categorical features without
    hardcoding, the dropdown options are now extracted dynamically from OHE columns
    and target encoder indexes.

11. Responsive layout and Three.js 3D integration
    Reason: To make the app compatible with mobile browsers, flex rows are wrapped
    under media queries. A Three.js wireframe watch is embedded inside a responsive column
    to create an interactive, eye-catching visual centerpiece.

13. COMPLETE PROJECT SUMMARY
------------------------------------------------------------
This Summer Internship project builds an end-to-end ML pipeline for
predicting watch prices from the WatchVine e-commerce dataset.

Starting from raw scraped data with 3,000+ columns, the pipeline:
1. Cleans data down to 1,488 rows x 18 meaningful columns
2. Performs comprehensive EDA with 25 visualizations
3. Engineers 129 features using one-hot and target encoding
4. Trains 6 regression models (Linear, Ridge, RF, XGBoost, LightGBM, CatBoost)
5. Evaluates all models on a held-out test set
6. Tunes the best model (XGBoost) with Optuna (100 trials)
7. Deploys as a Streamlit web app with real-time prediction

Final Model Performance:
  • Model: XGBoost (tuned)
  • R2 Score: 0.7117
  • RMSE: INR 466
  • MAE: INR 294

Key Challenge Solved: Resolved the 87.5% unknown brand and 89.1% unknown movement status
sparsity issues by implementing a text parser that extracts these attributes from the product titles,
improving R2 from 45% to 71.2%.

The Streamlit app allows users to input any watch attributes and
receive an instant price prediction with confidence range and
SHAP-based feature importance explanation.
